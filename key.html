TYPES OF KEYS

2nd JULY 2024 (Day-4)

Super Key

A super key is a set of one or more columns that can be used to uniquely identify a row in a table.

  • All candidate keys are super keys, but not all super keys are candidate keys.

Candidate Key

A candidate key is a column or a set of columns that can qualify as a unique key in the database.

  • There can be multiple candidate keys in a table.
  • One of the candidate keys is chosen as the primary key, while the others are called alternate keys.

Primary Key

A primary key is a unique identifier for a record in a table.

  • Each table can have only one primary key, which can consist of one or multiple columns.
  • Primary keys cannot contain NULL values.

Alternate Key

An alternate key is any candidate key that is not chosen as the primary key.

  • It is a unique key that could have been chosen as the primary key but was not

Artificial Key

An artificial key, also known as a surrogate key, is a unique identifier for a record in a database table that is not derived from application data.

  • It is typically generated automatically by the database system, such as an auto-incrementing number or a UUID.
  • Artificial keys are non-meaningful outside the database context and provide consistency and simplicity for managing relationships between tables.
  • They help improve performance and ensure data integrity by providing a stable and unique identifier.

Foreign Key

A foreign key is a column or a set of columns in a table that establishes a link between data in two tables.

  • It is a reference to the primary key in another table.
  • Foreign keys can contain NULL values.

Composite Key

A composite key is a primary key that consists of two or more columns.

  • It is used when a single column is not sufficient to uniquely identify records.

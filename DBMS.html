Database Management System(DBMS)

1st July 2024 (Day-3)

Data

Data refers to raw, unprocessed facts and figures collected from observations, measurements, or experiments. It can be in various forms such as numbers, text, images, or audio. Data is the foundation for information and knowledge when it is analyzed and interpreted.

Information

Information is data that has been processed, organized, or structured in a way that adds context and meaning, making it useful for analysis, decision-making, and communication. It transforms raw data into a format that is interpretable and valuable to its users.

Meta Data

Meta data is data that describes other data, providing information about a dataset's content, context, quality, structure, and management. It helps in understanding and managing the data effectively.

Data Dictionary

A data dictionary is a centralized repository that defines and describes the structure, relationships, and attributes of data within a database. It contains metadata about the data, including definitions, data types, constraints, and relationships between data elements, helping users and administrators understand and manage the data effectively.

Database

A database is an organized collection of structured information or data, typically stored electronically in a computer system, allowing for efficient retrieval, insertion, and management of data.

Table

A table is a collection of related data held in a structured format within a database, consisting of rows and columns where each row represents a record and each column represents a field.

Tuple(s)

Tuples are ordered lists of elements, often used to refer to a single row within a table in a relational database, representing a single record or data item.

constraint(s)

Constraints are rules enforced on data in a database to ensure accuracy and consistency, such as primary keys, foreign keys, unique constraints, and check constraints.

Domain(s)

Domains define the permissible values that a field (or attribute) in a database can hold, specifying the data type and range of values for a particular attribute.

Key(s)

Keys are attributes or sets of attributes used to uniquely identify records within a table, with primary keys uniquely identifying each record and foreign keys establishing relationships between tables.

Field(s)

Fields are individual pieces of data within a table, representing a specific attribute or property of a record, organized in columns and holding values of a particular data type.
